# Ideal customer profile (ICP) reference

> Canonical reference for `admin` skills.
> Owner: {{ORG_NAME}} Dewey admin.

This file defines who {{ORG_NAME}}'s ideal customer is. Skills that prepare for customer conversations, qualify accounts, or draft outbound messaging should load this file.

## Firmographic profile

The type of company we sell to best.

- **Company type:** (e.g., B2B SaaS, professional services firm, manufacturer)
- **Size:** (e.g., 50–500 employees, Series A–C, $10M–$100M ARR)
- **Industry:** (e.g., fintech, healthcare, logistics — list the top 2–3)
- **Geography:** (e.g., North America, EMEA, global)

## Buyer profile

The person who buys and the person who uses are often different.

- **Economic buyer:** (e.g., VP of Ops, CFO, RevOps lead) — writes the check
- **Champion:** (e.g., Senior Ops Manager) — drives the evaluation
- **End user:** (e.g., AEs, CS Managers) — uses the product daily

## Problem we solve

The specific, named pain the ICP has that {{ORG_NAME}} solves. Not a feature list — the pain they feel before they find us.

- *Pain 1:* What they're doing today and why it breaks down.
- *Pain 2:* A second common pain, if applicable.

## Non-ICP signals

Signals that tell us this is *not* a fit — so we stop early rather than six months in.

- *Non-fit signal 1:* (e.g., company below 20 employees)
- *Non-fit signal 2:* (e.g., requires on-premise deployment)

## How to use this file

Skills that draft outbound, prep for discovery calls, or qualify accounts should check the non-ICP signals explicitly. If a buyer matches a non-fit signal, surface it rather than ignoring it.
